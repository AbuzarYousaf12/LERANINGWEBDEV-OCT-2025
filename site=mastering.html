1. Introduction

Welcome to your comprehensive guide to mastering HTML and CSS! This guide is designed to take you from the basics all the way to building complex, responsive websites like Netflix. Whether you're a complete beginner or looking to refresh your knowledge, this guide has something for you.

By the end of this guide, you'll be able to:

  • Write semantic, well-structured HTML
  • Style websites with CSS using modern techniques
  • Create responsive layouts that work on all device sizes
  • Build a Netflix-like interface from scratch
  • Apply best practices for web development

To get the most out of this guide, I recommend following along by creating your own projects as you learn. The best way to master web development is through practice!

Note: This guide assumes you have a basic understanding of how to use a text editor and web browser. If you're completely new to web development, consider using Visual Studio Code as your editor.

2. HTML Basics

HTML (HyperText Markup Language) is the standard markup language for creating web pages. It describes the structure of a web page and consists of a series of elements that tell the browser how to display the content.

2.1 HTML Structure

Every HTML document follows a standard structure:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My First Web Page</title>
</head>
<body>
    <!-- Your content goes here -->
    <h1>Hello, World!</h1>
    <p>This is my first web page.</p>
</body>
</html>

Let's break down this structure:

  • <!DOCTYPE html>: Declares the document type and HTML version
  • <html>: The root element of an HTML page
  • <head>: Contains meta-information about the document
  • <meta>: Provides metadata about the HTML document
  • <title>: Specifies a title for the document
  • <body>: Contains the visible page content

2.2 HTML Elements

HTML elements are the building blocks of HTML pages. Here are some common HTML elements:

Headings

HTML provides six levels of headings, from <h1> (most important) to <h6> (least important):

<h1>This is a Heading 1</h1>
<h2>This is a Heading 2</h2>
<h3>This is a Heading 3</h3>
<h4>This is a Heading 4</h4>
<h5>This is a Heading 5</h5>
<h6>This is a Heading 6</h6>

Paragraphs

<p>This is a paragraph.</p>
<p>This is another paragraph.</p>

Links

<a href="https://www.example.com">This is a link</a>

Images

<img src="image.jpg" alt="Description of the image">

Lists

Unordered list:

<ul>
    <li>Item 1</li>
    <li>Item 2</li>
    <li>Item 3</li>
</ul>

Ordered list:

<ol>
    <li>First item</li>
    <li>Second item</li>
    <li>Third item</li>
</ol>

Divs and Spans

<div>This is a block-level container</div>
<span>This is an inline container</span>
Tip: <div> is a block-level element (starts on a new line), while <span> is an inline element (does not start on a new line).

2.3 HTML Attributes

HTML attributes provide additional information about HTML elements:

<tag attribute="value">Content</tag>

Common attributes:

  • id: Specifies a unique ID for an element
  • class: Specifies one or more class names for an element
  • style: Specifies an inline CSS style for an element
  • src: Specifies the URL of the media file (used with <img>, <audio>, <video>, etc.)
  • href: Specifies the URL of the page the link goes to
  • alt: Specifies an alternative text for an image

Example:

<a href="https://www.example.com" id="main-link" class="external-link" style="color: blue;">Visit Example.com</a>

2.4 Semantic HTML

Semantic HTML introduces meaning to the web page rather than just presentation. This helps search engines, screen readers, and other devices better understand your content.

Examples of semantic elements:

<header>...</header>
<nav>...</nav>
<main>...</main>
<article>...</article>
<section>...</section>
<aside>...</aside>
<footer>...</footer>

A semantic HTML structure might look like this:

<body>
    <header>
        <h1>My Website</h1>
        <nav>
            <ul>
                <li><a href="#">Home</a></li>
                <li><a href="#">About</a></li>
                <li><a href="#">Contact</a></li>
            </ul>
        </nav>
    </header>
    
    <main>
        <section>
            <h2>About Us</h2>
            <p>We are a company that...</p>
        </section>
        
        <article>
            <h2>Latest News</h2>
            <p>Our company has just launched...</p>
        </article>
        
        <aside>
            <h3>Related Links</h3>
            <ul>
                <li><a href="#">Link 1</a></li>
                <li><a href="#">Link 2</a></li>
            </ul>
        </aside>
    </main>
    
    <footer>
        <p>© 2023 My Company</p>
    </footer>
</body>
Tip: Using semantic HTML improves accessibility, SEO, and code readability.

3. CSS Basics

CSS (Cascading Style Sheets) is used to style and layout web pages. It describes how HTML elements should be displayed on screen, paper, or in other media.

There are three ways to insert CSS:

  • External CSS: Using a separate CSS file (recommended)
  • Internal CSS: Using a <style> element in the <head> section
  • Inline CSS: Using the style attribute in HTML elements

Example of external CSS:

/* styles.css */
body {
    font-family: Arial, sans-serif;
    margin: 0;
    padding: 20px;
    background-color: #f0f0f0;
}

h1 {
    color: #333;
    text-align: center;
}

Then link it in your HTML:

<head>
    <link rel="stylesheet" href="styles.css">
</head>

3.1 CSS Selectors

CSS selectors are used to "find" (or select) the HTML elements you want to style.

Basic Selectors

Selector Example Description
Element p { } Selects all <p> elements
ID #header { } Selects the element with id="header"
Class .btn { } Selects all elements with class="btn"
Universal * { } Selects all elements

Combinators

Selector Example Description
Descendant div p { } Selects all <p> elements inside <div> elements
Child div > p { } Selects all <p> elements where the parent is a <div> element
Adjacent Sibling div + p { } Selects the first <p> element that is placed immediately after <div> elements
General Sibling div ~ p { } Selects all <p> elements that are siblings of <div> elements

Pseudo-classes and Pseudo-elements

/* Pseudo-classes */
a:hover { color: red; }          /* When user hovers over a link */
button:focus { outline: none; }  /* When button has focus */
li:first-child { font-weight: bold; } /* First item in a list */

/* Pseudo-elements */
p::first-line { font-weight: bold; }  /* First line of a paragraph */
p::before { content: "→ "; }          /* Insert content before each paragraph */
p::after { content: " ←"; }           /* Insert content after each paragraph */

3.2 Box Model

The CSS box model is essentially a box that wraps around every HTML element. It consists of: margins, borders, padding, and the actual content.

.box {
    /* Content dimensions */
    width: 300px;
    height: 200px;
    
    /* Padding */
    padding-top: 20px;
    padding-right: 30px;
    padding-bottom: 20px;
    padding-left: 30px;
    /* Shorthand: padding: 20px 30px; */
    
    /* Border */
    border-width: 2px;
    border-style: solid;
    border-color: #333;
    /* Shorthand: border: 2px solid #333; */
    
    /* Margin */
    margin-top: 10px;
    margin-right: 15px;
    margin-bottom: 10px;
    margin-left: 15px;
    /* Shorthand: margin: 10px 15px; */
}
Tip: Use box-sizing: border-box; to include padding and border in an element's total width and height. 
* {
    box-sizing: border-box;
}

3.3 Typography

CSS provides properties to control text appearance:

body {
    font-family: 'Arial', sans-serif;  /* Font family */
    font-size: 16px;                   /* Font size */
    font-weight: normal;               /* Font weight (normal, bold, etc.) */
    font-style: normal;                /* Font style (normal, italic, etc.) */
    line-height: 1.5;                  /* Line height */
    letter-spacing: 0.5px;             /* Space between letters */
    word-spacing: 2px;                 /* Space between words */
    text-align: left;                  /* Text alignment (left, right, center, justify) */
    text-decoration: none;             /* Text decoration (none, underline, etc.) */
    text-transform: none;              /* Text transform (none, uppercase, lowercase, capitalize) */
}

Web Fonts

You can use custom fonts using @font-face or by importing from services like Google Fonts:

/* Using Google Fonts */
@import url('https://fonts.googleapis.com/css2?family=Roboto:wght@400;700&display=swap');

body {
    font-family: 'Roboto', sans-serif;
}

3.4 Colors and Backgrounds

CSS provides several ways to specify colors:

/* Color names */
color: red;

/* Hex values */
color: #ff0000;  /* Same as red */
color: #f00;     /* Shorthand */

/* RGB values */
color: rgb(255, 0, 0);  /* Same as red */

/* RGBA (with alpha/transparency) */
color: rgba(255, 0, 0, 0.5);  /* Semi-transparent red */

/* HSL (Hue, Saturation, Lightness) */
color: hsl(0, 100%, 50%);  /* Same as red */

/* HSLA (with alpha/transparency) */
color: hsla(0, 100%, 50%, 0.5);  /* Semi-transparent red */

Background Properties

.element {
    background-color: #f0f0f0;
    background-image: url('background.jpg');
    background-repeat: no-repeat;  /* no-repeat, repeat, repeat-x, repeat-y */
    background-position: center;   /* top, right, bottom, left, center or x y coordinates */
    background-size: cover;        /* auto, cover, contain, or specific values */
    
    /* Shorthand */
    background: #f0f0f0 url('background.jpg') no-repeat center / cover;
}

Gradients

/* Linear Gradient */
background: linear-gradient(to right, #ff0000, #0000ff);

/* Radial Gradient */
background: radial-gradient(circle, #ff0000, #0000ff);

4. CSS Layout

CSS provides several techniques for controlling the layout of web pages:

4.1 Display Property

The display property specifies if/how an element is displayed:

/* Common display values */
.element {
    display: block;       /* Takes up the full width available */
    display: inline;      /* Takes only as much width as necessary */
    display: inline-block; /* Like inline but can have width and height */
    display: none;        /* Removes the element completely */
    display: flex;        /* Creates a flex container */
    display: grid;        /* Creates a grid container */
}

4.2 Positioning

The position property specifies the type of positioning method used for an element:

.element {
    /* Position types */
    position: static;    /* Default - follows normal document flow */
    position: relative;  /* Positioned relative to its normal position */
    position: absolute;  /* Positioned relative to nearest positioned ancestor */
    position: fixed;     /* Positioned relative to the viewport */
    position: sticky;    /* Positioned based on scroll position */
    
    /* Used with non-static positioning */
    top: 10px;
    right: 20px;
    bottom: 30px;
    left: 40px;
    z-index: 1;  /* Controls stacking order */
}
Tip: When using position: absolute, make sure the parent element has position: relative if you want to position relative to the parent.

4.3 Float and Clear

The float property is used for positioning and formatting content:

.left {
    float: left;   /* Element floats to the left */
}

.right {
    float: right;  /* Element floats to the right */
}

.clearfix::after {
    content: "";
    display: table;
    clear: both;   /* Clears floats */
}
Note: While floats are still used, modern layouts typically use Flexbox or Grid instead.

4.4 Flexbox

Flexbox is a one-dimensional layout method for laying out items in rows or columns:

.container {
    display: flex;                /* Creates a flex container */
    flex-direction: row;          /* row, row-reverse, column, column-reverse */
    flex-wrap: wrap;              /* nowrap, wrap, wrap-reverse */
    flex-flow: row wrap;          /* Shorthand for flex-direction and flex-wrap */
    justify-content: space-between; /* Alignment along main axis */
    align-items: center;          /* Alignment along cross axis */
    align-content: space-around;  /* Alignment of flex lines (when multiple lines) */
    gap: 10px;                    /* Gap between flex items */
}

.item {
    flex-grow: 1;     /* Ability to grow relative to other items */
    flex-shrink: 0;   /* Ability to shrink relative to other items */
    flex-basis: auto; /* Default size before remaining space is distributed */
    flex: 1 0 auto;   /* Shorthand for grow, shrink, basis */
    align-self: flex-end; /* Override align-items for specific item */
    order: 2;         /* Controls order of items */
}

Common Flexbox Patterns

Centering content:

.centered {
    display: flex;
    justify-content: center;  /* Horizontal centering */
    align-items: center;      /* Vertical centering */
    height: 100vh;            /* Full viewport height */
}

Creating a responsive navigation:

.navbar {
    display: flex;
    justify-content: space-between;
}

.nav-links {
    display: flex;
    gap: 20px;
}

@media (max-width: 768px) {
    .navbar {
        flex-direction: column;
    }
}

4.5 CSS Grid

CSS Grid is a two-dimensional layout system designed for complex layouts:

.container {
    display: grid;
    grid-template-columns: 1fr 2fr 1fr;  /* Three columns with proportional widths */
    grid-template-rows: 100px auto 100px;  /* Three rows with specified heights */
    grid-template-areas:  /* Named grid areas */
        "header header header"
        "sidebar content content"
        "footer footer footer";
    grid-gap: 10px;  /* Gap between grid items */
    /* Or use column-gap and row-gap separately */
}

.header { grid-area: header; }
.sidebar { grid-area: sidebar; }
.content { grid-area: content; }
.footer { grid-area: footer; }

/* Positioning items individually */
.item {
    grid-column: 1 / 3;  /* Start at column line 1, end at column line 3 */
    grid-row: 2 / 4;     /* Start at row line 2, end at row line 4 */
    /* Or use grid-column-start, grid-column-end, grid-row-start, grid-row-end */
}

Responsive Grid

.responsive-grid {
    display: grid;
    grid-template-columns: repeat(auto-fill, minmax(250px, 1fr));
    gap: 20px;
}
Tip: The repeat(auto-fill, minmax(250px, 1fr)) pattern creates a responsive grid where items are at least 250px wide and as many fit as possible in each row.

5. Responsive Web Design

Responsive web design makes web pages look good on all devices. The design should adapt to different screen sizes and viewports.

5.1 Media Queries

Media queries allow you to apply CSS styles based on various conditions, particularly the width of the browser window:

/* Base styles for all devices */
body {
    font-size: 16px;
}

/* Styles for tablets */
@media (max-width: 768px) {
    body {
        font-size: 14px;
    }
    .container {
        width: 90%;
    }
}

/* Styles for mobile phones */
@media (max-width: 480px) {
    body {
        font-size: 12px;
    }
    .container {
        width: 100%;
    }
    .hide-on-mobile {
        display: none;
    }
}

Media queries can also target other features:

/* Target devices with a minimum height */
@media (min-height: 800px) { ... }

/* Target devices in landscape orientation */
@media (orientation: landscape) { ... }

/* Target devices with high-resolution displays */
@media (min-resolution: 2dppx) { ... }

/* Combine multiple conditions */
@media (min-width: 768px) and (max-width: 1024px) { ... }

5.2 Mobile-First Approach

Mobile-first means designing for mobile devices first, then progressively enhancing for larger screens:

/* Base styles for mobile devices */
.container {
    width: 100%;
    padding: 10px;
}

.nav {
    display: flex;
    flex-direction: column;
}

/* Enhance for tablets */
@media (min-width: 768px) {
    .container {
        width: 750px;
        margin: 0 auto;
    }
    
    .nav {
        flex-direction: row;
    }
}

/* Enhance for desktops */
@media (min-width: 1024px) {
    .container {
        width: 980px;
    }
}
Tip: Mobile-first is generally preferred because:
  1. It forces you to prioritize content
  2. It typically results in cleaner code
  3. It often performs better on mobile devices

5.3 Responsive Images

Make images responsive by using CSS:

img {
    max-width: 100%;
    height: auto;
}

For more advanced responsive images, you can use the srcset and sizes attributes:

<img src="small.jpg"
     srcset="small.jpg 500w,
             medium.jpg 1000w,
             large.jpg 1500w"
     sizes="(max-width: 600px) 100vw,
            (max-width: 1200px) 50vw,
            33vw"
     alt="Responsive image">

You can also use the picture element:

<picture>
    <source media="(max-width: 600px)" srcset="small.jpg">
    <source media="(max-width: 1200px)" srcset="medium.jpg">
    <img src="large.jpg" alt="Responsive image">
</picture>

5.4 Responsive Typography

Make your typography responsive using relative units:

html {
    font-size: 16px;  /* Base font size */
}

h1 {
    font-size: 2rem;  /* 2 times the root font size */
}

p {
    font-size: 1rem;  /* Same as root font size */
}

.small-text {
    font-size: 0.8rem;  /* 80% of root font size */
}

@media (max-width: 768px) {
    html {
        font-size: 14px;  /* Smaller base font on mobile */
    }
}

Using viewport units for fluid typography:

h1 {
    font-size: calc(1.5rem + 2vw);  /* Scales with viewport width */
}

.hero-text {
    font-size: 5vmin;  /* Uses the smaller of viewport width or height */
}

6. Advanced CSS

6.1 Transitions

CSS transitions allow you to change property values smoothly over a specified duration:

.button {
    background-color: blue;
    color: white;
    padding: 10px 20px;
    border-radius: 4px;
    /* Define the transition */
    transition-property: background-color, transform;
    transition-duration: 0.3s;
    transition-timing-function: ease-in-out;
    transition-delay: 0s;
    /* Shorthand */
    transition: background-color 0.3s ease-in-out, transform 0.3s ease-in-out;
}

.button:hover {
    background-color: darkblue;
    transform: scale(1.05);
}
Tip: Common properties to transition: color, background-color, opacity, transform, width, height, and margin.

6.2 Animations

CSS animations allow elements to gradually change from one style to another:

/* Define the keyframes */
@keyframes slide-in {
    0% {
        transform: translateX(-100%);
        opacity: 0;
    }
    100% {
        transform: translateX(0);
        opacity: 1;
    }
}

/* Apply the animation */
.animated-element {
    animation-name: slide-in;
    animation-duration: 1s;
    animation-timing-function: ease-out;
    animation-delay: 0s;
    animation-iteration-count: 1;
    animation-direction: normal;
    animation-fill-mode: forwards;
    animation-play-state: running;
    
    /* Shorthand */
    animation: slide-in 1s ease-out 0s 1 normal forwards running;
}

Multiple animations:

.multi-animated {
    animation: 
        slide-in 1s ease-out,
        fade-in 2s ease;
}

6.3 CSS Variables (Custom Properties)

CSS variables allow you to store and reuse values throughout your stylesheet:

:root {
    /* Define variables in the root scope */
    --primary-color: #3498db;
    --secondary-color: #2ecc71;
    --font-size-base: 16px;
    --spacing-unit: 8px;
}

.button {
    /* Use variables */
    background-color: var(--primary-color);
    color: white;
    padding: calc(var(--spacing-unit) * 2) calc(var(--spacing-unit) * 3);
    font-size: var(--font-size-base);
}

.button.secondary {
    background-color: var(--secondary-color);
}

/* Change variables in media queries */
@media (max-width: 768px) {
    :root {
        --font-size-base: 14px;
        --spacing-unit: 6px;
    }
}
Tip: CSS variables are especially useful for theme switching (like dark mode).

6.4 CSS Preprocessors

CSS preprocessors like Sass, Less, and Stylus extend CSS with features like variables, nesting, mixins, and functions:

Sass Example

// Variables
$primary-color: #3498db;
$secondary-color: #2ecc71;
$border-radius: 4px;

// Mixins
@mixin flex-center {
    display: flex;
    justify-content: center;
    align-items: center;
}

@mixin button($bg-color) {
    background-color: $bg-color;
    color: white;
    padding: 10px 20px;
    border-radius: $border-radius;
    
    &:hover {
        background-color: darken($bg-color, 10%);
    }
}

// Usage
.container {
    max-width: 1200px;
    margin: 0 auto;
    
    // Nesting
    .header {
        @include flex-center;
        
        .logo {
            font-size: 2rem;
            color: $primary-color;
        }
        
        .nav {
            ul {
                display: flex;
                list-style: none;
                
                li {
                    margin: 0 10px;
                }
            }
        }
    }
}

.button {
    @include button($primary-color);
    
    &.secondary {
        @include button($secondary-color);
    }
}
Note: While preprocessors are still popular, many of their features (like variables) are now available in standard CSS. For small projects, you might not need a preprocessor.

<header class="netflix-header">

9. Best Practices

As you continue to develop your HTML and CSS skills, keep these best practices in mind:

HTML Best Practices

  • Use semantic HTML: Choose the right elements for your content (e.g., <article>, <section>, <nav>).
  • Maintain proper indentation: Keep your code readable with consistent indentation.
  • Use meaningful class and ID names: Choose names that reflect the purpose of the element, not its appearance.
  • Optimize images: Use appropriate image formats and sizes to improve page load times.
  • Validate your HTML: Use the W3C Validator to ensure your code is error-free.
  • Include proper meta tags: Add description, viewport, and other important meta tags.
  • Ensure accessibility: Use alt attributes for images, proper ARIA attributes, and semantic structure.

CSS Best Practices

  • Follow a naming convention: Consider using methodologies like BEM (Block, Element, Modifier).
  • Organize your CSS: Group related styles and use comments to separate sections.
  • Minimize specificity: Avoid overly specific selectors that can cause issues later.
  • Use shorthand properties: For example, use margin: 10px 20px; instead of setting each side separately.
  • Avoid !important: Use it sparingly and only when absolutely necessary.
  • Use relative units: Prefer rem, em, %, and vh/vw over pixels for better responsiveness.
  • Implement mobile-first design: Start with styles for small screens, then add complexity for larger screens.
  • Minimize HTTP requests: Combine CSS files and use CSS sprites for icons when appropriate.
  • Use CSS preprocessors wisely: Don't overcomplicate your code with unnecessary nesting or variables.

Performance Best Practices

  • Minimize CSS and JS files: Remove unnecessary whitespace and comments in production.
  • Optimize images: Compress images and use modern formats like WebP when possible.
  • Lazy load images: Load images only when they're about to enter the viewport.
  • Use CSS transitions instead of JS animations: When possible, for better performance.
  • Avoid render-blocking resources: Load CSS in the head and JS before the closing body tag.
  • Consider using CSS frameworks wisely: Only include the components you need.
